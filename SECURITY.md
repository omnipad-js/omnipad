# Security Policy

## Supported Versions

As this is a community project maintained in our spare time, security updates are currently provided only for the latest release.

| Version | Supported          |
| ------- | ------------------ |
| latest  | :white_check_mark: |
| < 0.5.0 | :x:                |

## Reporting a Vulnerability

**Please do not open a public issue for security vulnerabilities.**

If you discover a potential security risk in this project, please use the **GitHub Private Vulnerability Reporting** feature:

1. Navigate to the main page of the repository.
2. Under the repository name, click **Security**.
3. On the left sidebar, under "Reporting", click **Advisories**.
4. Click **Report a vulnerability**.

### Our Response Logic

- **Acknowledgment**: We will acknowledge your report as soon as possible (usually within a week).
- **Maintenance Status**: Please note that we are currently only performing essential maintenance and bug fixes. Security patches will be handled on a "best-effort" basis.
- **Disclosure**: After a fix is confirmed and released, a security advisory will be published to credit your discovery.

Thank you for helping keep this project safe for the community!
